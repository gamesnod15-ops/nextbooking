# RandevumKolay — Uygulama Geliştirme Kuralları

## Backend
- Clean Architecture: Domain → Application → Infrastructure → API
- CQRS with MediatR — commands/queries are separate
- All commands/queries must have FluentValidation validators
- Use `ICurrentTenantService` to filter all tenant data — never skip tenant isolation
- Domain entities must use private setters and factory methods (`Create()`)
- Use `Result<T>` pattern for expected failures; throw exceptions for unexpected ones

## Kod Standartları
- snake_case for database columns (EFCore.NamingConventions)
- RefreshToken rotation: always revoke old token when issuing new
- Passwords: only BCrypt via `IPasswordHasher` — never in Application layer directly
- Never log passwords, tokens, or PII

## Frontend
- Admin Panel: Vite + React + TypeScript + Tailwind + Redux Toolkit + TanStack Query
- Booking Frontend: Next.js 15 App Router + TypeScript + Tailwind + TanStack Query
- Use TanStack Query for all server state; Redux only for client state (auth, UI)
- Token refresh via axios interceptor (silent refresh pattern)

## İnfrastrüktur
- Docker Compose for production deployment
- Nginx handles subdomain routing — each tenant gets their own subdomain
- Redis TTL: 5 minutes for tenant lookups, 15 minutes for general cache
- Always use health checks in Docker services
